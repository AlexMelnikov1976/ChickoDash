### 3. Исправлен `add-user.js`
Скрипт раньше падал в любом окружении без глобально установленного wrangler 3.x.

| Что | Было | Стало |
|---|---|---|
| Команда wrangler | `wrangler kv:key put` | `npx wrangler kv key put` |

- Синтаксис `kv:key` устарел в wrangler 4.x — теперь только `kv key` через пробел.
- Префикс `npx` нужен, потому что `wrangler` не установлен глобально.
- Проверено: тестовый запуск с `wrangler-syntax-test@example.invalid` → запись прошла → удалена.

### 4. Обновлён `docs/USERS.md`

- Добавлена запись о новом пользователе.

### 5. Миграция mixed-case ключа в KV
Обнаружена запись `user:Chicko.TSK@gmail.com` (заглавные буквы), при том что сервер ищет ключ в lowercase (`src/magic_link.ts:69` — `email.trim().toLowerCase()`). Пользователь физически не мог войти.

- Создан новый ключ `user:chicko.tsk@gmail.com` с сохранением исходного `user_id` (`Chicko.TSK@gmail.com`) для непрерывности логов в `chicko.user_activity_log`.
- В значение добавлено поле `note` с указанием на миграцию.
- Старый ключ удалён.
- Общий count в KV = 33 (без изменений).
