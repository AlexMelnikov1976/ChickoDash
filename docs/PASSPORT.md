## 1. Что это и зачем

**Chicko Analytics** — аналитическая платформа для франчайзи сети ресторанов Chicko. Показывает ключевые метрики, сравнивает каждый ресторан с сетью, строит динамику, выдаёт рекомендации.

**Пользователи:** 42 франчайзи сети Chicko + управляющая компания.

**Текущий фокус:** **Фаза 1 полировки** по результатам AI-критики + обратной связи владельца.

## 5. Архитектурные решения

### 5.1–5.33 — предыдущие (см. v3.15)

### 5.34 Top-down UX — прогноз сверху, детали ниже (19.04.2026)

**Решение:** Первый экран дашборда перестраивается: сначала прогноз «куда идём», потом детали. Руководителю не нужно проваливаться в 6 KPI если всё в норме.

**Причина:** Цитата владельца: «Меня как руководителя в первую очередь интересует прогноз куда мы идём, а уже потом средняя выручка и средний чек. Если у меня всё хорошо — зачем мне проваливаться в детали?»

**Статус:** Идея зафиксирована, реализация — задача #71 в Фазе 1.4.

### 5.35 Like-for-like сравнения по дням недели (20.04.2026)

**Решение:** Все сравнения в дашборде используют **like-for-like** логику по дням недели. Средний чек за субботу сравнивается с нормой суббот (за 90 дней), а не с усреднённой неделей.

**Три уровня сравнения:**
1. **Ваша норма** (основная база) — медиана текущего ресторана за те же дни недели за 90 дней. Показывается если история ≥ 14 дней. Это то, чем владелец реально управляет.
2. **Медиана сети** (контекст) — медиана всех 43 ресторанов за те же дни недели за 90 дней. Fallback если «нормы» нет.
3. **Топ-25% сети** (потенциал) — 75-й перцентиль. Показывает куда можно расти.

**Причина:** Инсайт владельца: «Сравнивать средние чеки за субботу и за понедельник неверно». Это фундаментальная ошибка предыдущей версии.

**Реализация:** 2 SQL-запроса при загрузке (`loadDowProfiles`), функция `dowBenchmarks(ts)` агрегирует профили по DOW текущего периода.

**Что это дало:**
- «Низкий средний чек» (при чеке 1439₽ vs P75 1970₽) → «Средний чек в вашей норме» (vs своя норма 1448₽, разница -0.6%). Честно и не паникует.
- Доставка 26.5% при медиане сети 15.9% → не «хуже сети», но «хуже вашей нормы 32.3%» → актуальная проблема.

### 5.36 Виджет обратной связи — сквозная автоматизация (20.04.2026)

**Решение:** В дашборде добавлена кнопка 💬 (floating, правый нижний угол). Клик открывает модалку с 4 категориями (Баг / Идея / Данные неверны / Непонятно) + текстовое поле. Фидбек уходит:

```
Дашборд → POST /api/feedback (Worker, JWT auth)
  → waitUntil → n8n webhook (fire-and-forget)
    → Notion (база «Обратная связь», Kanban по статусам)
    → Telegram (уведомление в чат -1003396450964)
```

**Причина:** Франчайзи начинают пользоваться дашбордом. Нужен канал обратной связи без барьеров — прямо из интерфейса, без email/WhatsApp/устных просьб.

**Notion база:** Колонки: Тема, Категория, Статус (Новый/В работе/Готово/Отклонён), Пользователь, Ресторан, Приоритет (P0-P3), Получено. Kanban-вьюшка по статусам.

**Secrets:** `FEEDBACK_WEBHOOK` (n8n Production URL). Notion API key — в n8n credentials (HTTP Header Auth).

### 5.37 Прогноз выручки — Алгоритм Г с YoY-коррекцией (20.04.2026)

**Решение:** Блок прогноза на месяц — первый элемент на главной, выше KPI. Алгоритм Г (гибрид):

1. ≥7 дней данных в текущем месяце → DOW-медианы текущего месяца (самый точный)
2. <7 дней + есть прошлый год → DOW прошлого года × коэффициент YoY (k = выручка последнего полного месяца / тот же месяц год назад)
3. Fallback → 90-дневные DOW-профили (`MY_DOW`/`NET_DOW`)

**Сезонность:** Коэффициент YoY автоматически корректирует данные прошлого года (сезон март–август vs несезон сентябрь–февраль). К 8-му дню месяца переключается на реальные данные.

**Галочка «Вся сеть»:** Переключатель в хедере (`toggleNetworkView`). При включении R = `buildNetworkR()` (агрегат всех ресторанов: суммы выручек, взвешенные %, средний чек = выручка/чеки). Все блоки страницы (KPI, алерты, инсайты, прогноз) пересчитываются.

**Причина:** Цитата владельца: «Меня как руководителя в первую очередь интересует прогноз куда мы идём».
